- E1.1 — 3D Scene Architecture (React Three Fiber)

## 1. Goal

To provide a high-fidelity 3D visualization of the experiment results, allowing users to inspect the collision from any angle, verify camera placement, and visually confirm the accuracy of the 3D reconstruction.

## 2. Technical Stack

| Concern               | Library                 | Reason                                                     |
| --------------------- | ----------------------- | ---------------------------------------------------------- |
| **3D Engine**         | Three.js                | Industry standard for web-based 3D.                        |
| **React Integration** | React Three Fiber (R3F) | Declarative 3D, perfect for state-driven updates.          |
| **Utilities**         | @react-three/drei       | Helpers for cameras, controls, and basic shapes.           |
| **Physics State**     | Zustand                 | Existing stores (`resultsStore`) will drive the animation. |
| **Interpolation**     | GSAP or lerp            | Smooth 60fps playback from 30fps data.                     |

## 3. Coordinate System & Scene Setup

### 3.1 1:1 Mapping

The 3D scene will map 1:1 with the coordinate system defined in **Module E**:

- **Origin (0,0,0):** Optical center of Camera 0.
- **Scale:** 1 unit = 1 metre.
- **Axes:**
  - **X:** Horizontal (Right)
  - **Y:** Vertical (Up)
  - **Z:** Depth (Into scene)
- **Handedness:** Right-handed (Native to both OpenCV and Three.js).

### 3.2 Global Elements

- **GridHelper:** 10m x 10m grid on the X-Z plane (at Y=0 or slightly offset to represent the "floor").
- **Lights:** One AmbientLight for base visibility, one DirectionalLight to give the balls volume/shadows.
- **Controls:** `OrbitControls` (Drei) to allow the user to pan/zoom/rotate around the collision point.

## 4. Visual Components

### 4.1 Virtual Cameras (The "Phones")

- Render two 3D models (or simple BoxGeometries) representing the smartphones.
- **Phone 0:** Placed at `(0,0,0)` with identity rotation.
- **Phone 1:** Placed at `T` (translation vector) with rotation `R` from `stereo_extrinsics.json`.
- **Frustums:** Optional dashed lines showing the field of view of each camera to confirm the "overlap" zone.

### 4.2 The Balls

- **Geometry:** `SphereGeometry` with a radius matching the physical ball (e.g., 20mm -> 0.02 units).
- **Material:** Distinct colors per `ball_id` (matched to 2D trajectory colors).
- **Position:** Updated every frame via the `useFrame` hook from R3F.

### 4.3 Trajectory Trails

- **Static Trail:** A `Line` or `CatmullRomCurve3` showing the entire path from frame 0 to N.
- **Active Trail:** A "comet tail" effect that follows the ball during playback, fading out after a few frames.

### 4.4 Uncertainty & Quality (The "Science View")

- **Uncertainty Volume:** A semi-transparent sphere around each ball. Its radius is driven by `max(X_unc, Y_unc, Z_unc)`. In areas of high noise, the sphere grows.
- **Quality Alert:** If `flagged == true` for a frame (high reprojection error), the ball's material pulses **red**.

## 5. Animation & Sync Logic

### 5.1 Playback Synchronization

The 3D view must be perfectly synced with the **2D Frame Scrubber** and **Velocity Charts**.

```typescript
// Pseudocode for frame sync
const currentFrame = useResultsStore((state) => state.currentFrame);
const trajectory = useResultsStore(
  (state) => state.physicsResult.balls[0].trajectory3D,
);

useFrame((state, delta) => {
  if (isPlaying) {
    // Interpolate between currentFrame and currentFrame + 1 for 60fps smoothness
    const targetPos = interpolate(
      trajectory[currentFrame],
      trajectory[currentFrame + 1],
      subFrameProgress,
    );
    ballRef.current.position.set(targetPos.X, targetPos.Y, targetPos.Z);
  } else {
    // Jump to exact frame position
    const pos = trajectory[currentFrame];
    ballRef.current.position.set(pos.X, pos.Y, pos.Z);
  }
});
```

## 6. Data Requirements (Protocol Changes)

The `PhysicsResult` gRPC message and `positions_3d.json` must include the full array of points:

```protobuf
message Point3D {
  double x = 1;
  double y = 2;
  double z = 3;
  double x_unc = 4;
  double y_unc = 5;
  double z_unc = 6;
  bool flagged = 7;
}

message BallResult {
  ...
  repeated Point3D trajectory_3d = 10;
}
```

## 7. Implementation Steps

### Step 1: Frontend Boilerplate

- Install `three`, `@types/three`, `@react-three/fiber`, `@react-three/drei`.
- Create a `ThreeDScene.tsx` component with a basic Canvas and GridHelper.
- Embed the Canvas in the **ResultsPage.tsx**.

### Step 2: Camera/Phone Rendering

- Load `stereo_extrinsics.json`.
- Place two BoxGeometries in the scene representing the devices.
- **Validation:** Ensure Phone 1 is correctly positioned and rotated relative to Phone 0.

### Step 3: Ball & Trajectory

- Map the `trajectory_3d` array to a series of `Vector3` points.
- Implement the `Ball3D` component that moves based on the current scrubber index.
- Add a `Line` component for the trajectory trail.

### Step 4: Polish & Interaction

- Add `OrbitControls`.
- Implement uncertainty ellipsoids.
- Add a "Follow Ball" toggle that pins the OrbitControls target to the moving ball.
